 # **Работа с GIT**

***Команды Git***

## 1. Проверка наличия установленного Git
В терминале выполнить команду 

`git version`

 Если Git установлен, появится сообщение с версией программы. Иначе появится сообщение о ошибке.

 ## 2. Установка Git
 Загружаем последнюю версию Git c сайта:

 http://git-scm.com/downloads

 Устанавливаем с настройками по умолчанию.

## 3. Настройка Git
При первом использоваение Git необхолдимо представиться.

Для этого нужно ввести в терминале две команды:

```
git config --global user.name "Ваше имя"
git config --global user.email "Почта@example.com"
``` 

## 4. Cоздание репозитория
Получить репозиторий можно двумя способами:
1. В терминале переходим к папке, в которой хотим создать репозиторий. Выполняем команду
```
git init
```
При инициализации Git создаст скрытую папку. В ней содержатся все объекты и ссылки, которые Git использует и создаёт в истории работы над проектом.

2. Cкопировать существующий репозиторий, выполнив команду
```
git clone "адрес репозитория"
```

## 5. Получение от Git информации о его текущем состоянии
Просмотреть статус нужного репозитория можно по ключевому слову `status` : его действие распространяется на подготовленные, неподготовленные и неотслеживаемые файлы.
Для получения информации необходимо выполнить команду
```
git status
```

## 6. Фиксация изменений
Для сохранения изменений, их необходимо закоммитить. 
Сперва необходимо обозначить эти файлы для Git, при помощи команды 
`git add` добавляющей их к коммиту. Добавлять их можно по отдельности:
```
git add "имя файла"
```
или все сразу:
```
git add .
```
Для создания самого коммита, необходимо ввести команду
```
git commit -m"имя коммита"
```
## 7. Просмотр изменений
Для просмотра списка всех фиксаций, а так же сведений о них (автора, списка изменений и даты, когда они были сделаны), необходимо ввести команду 
```
git log
```
Кроме того, добавив флаг -p, вы можете подробно изучить изменения, внесённые в каждый файл.
```
git log -p
```

Для просмотра разницы между текущим и закоммиченым файлом, используется команда 
```
git diff
```
По умолчанию отображаются только изменения, не подготовленные для фиксации.


## 8. Переход от одного коммита к другому

для перемещения между фиксациями нужно ввести команду
```
git checkout "персональный идентификатор коммита"
```
Идентификатор (достаточно 4-х символов), как и другие сведения, можно посмотреть м помощью `git log` (см. раздел 7), или же для удобства использовать 
```
git log --oneline
```
После просмотра фиксаций необходимо вернуться в актуальное состояние. Для возвращения к актуальному состоянию нужно ввести команду 
```
git switch -
```
или
```
git checkout master
```
## 9. Ветки

**Создание новой ветки** 

Создать новую ветку можно с помощью параметра branch, указав имя ветки.
```
git branch new_branch_name
```

Но Git не переключится на неё автоматически. Для  перехода нужно ввести
``` 
 git checkout "имя ветки"
 ```
 или, для автоматоического перехода
 
 ```
git checkout -b "имя ветки"
```

 
Можно просматривать полный список веток, используя параметр branch. 
Команда отобразит 
все ветки, отметит текущую звёздочкой (*) и выделит её цветом.

```
git branch
```
Если вы завершили работу над веткой и объединили её с основной, можно её с основной, 
можно её удалить без потери истории.

 Удалить ветку можно параметром branch с добавлением флага `-d` и указанием имени 
ветки. 
 
 

```
git branch -d "имя ветки"
```

Однако, если выполнить команду удаления до слияния — в результате появится сообщение 
об ошибке. 

Для принудительного удаления ветки используется флаг -D с заглавной буквой. В этом 
случае ветка будет удалена независимо от текущего статуса, без предупреждений.

```
git branch -D existing_branch_name
```

Объединить две ветки можно параметром `merge` с указанием имени ветки.

 Команда объединит указанную ветку с основной.

```
git merge "имя ветки"
```

Указанная команда объединит заданную ветку с основной и произведёт коммит слияния. Это необходимо для фиксации всех слияний в вашем репозитории.

В процессе слияния может произойти конфликт и `git` предложит четыре варианта решения конфликта:

`accept current change` - принять текущую версию

`accept incoming change` - принять вливаемую версию

`accept both changes` - оставить обе версии

`compare changes` - сравнить их

После выбора одного из действий необходимо закоммитить изменения.

# ***Cправочник по markdown***
## 1. Выделение текста
Текст можно выделить **жирным**, заключив его между парами звездочек или двойными нижними подчеркиваниями:
```
**Жирный**
__Жирный__
```
Выделение *курсивом* осуществляется заключением текста между звездочками  нижними поркями:
```
*Курсив*
_Курсив_
```
~~Зачеркнутый~~ текст заключается между парами тильд:
```
~~Зачеркнуто~~
```

Способы выделения можно комбинировать

## 2. Cписки и отступы

Чтобы оформить строку в элемент маркированного списка, в начале нужно поставить плюс, минус или звездочку. Звездочка не приведет к курсивному выделению, потому что отделяется от слова пробелом.
```
- Пункт 1

- Пункт 2

- Пункт 3


или


+ Пункт 1

+ Пункт 2

+ Пункт 3


или


* Пункт 1

* Пункт 2

* Пункт 3
```
Если необходимо создать нумерованный список, используйте в начале строки цифру с точкой. 
```
1. Пункт 1

2. Пункт 2

3. Пункт 3
```
## 3. Ccылки 
Чтобы поставить гиперссылку без анкора, нужно взять URL в угловые скобки. С e-mail – аналогично.

Если вставлять с анкором, то тогда текст ссылки заключается в квадратные скобки, а адрес страницы – в круглые. Рядом с URL можно прописать тайтл, его объявляют в кавычках (он тоже остается внутри круглых скобок).

## 4. Цитаты и вставки кода
Если безанкорные ссылки оформляются двумя угловыми скобками, то для цитаты нужна только одна такая скобка. Все очень просто:

> Привет! Это цитата

>> Это тоже цитата

Цитаты вкладываются как в списки, так и в другие цитаты. Цитаты также могут включать в себя заголовки, списки и код.

Код можно как внутрь строк, так и отдельными блоками. Для соответствующей разметки используют грависы или обратные тики. Инлайн-код выделяют одиночными символами, а блоки – тройными. При оформлении целого блока можно указать язык программирования, чтобы подсветить соответствующий синтаксис

Например:
Код на JavaScript:

```javascript

let greeting1 = 'Father!';

console.log(greeting1);

let greeting2 = 'Mother!';

console.log(greeting2);

```
## 5. Картинки
Чтобы добавить изображения в текст, достаточно ввести следующее:
```
![имя файла](адрес файла)
```
например:

![GitHub](alDEUUYb6Ws.jpg)

Чтобы не добавлять файл с изображением в **отслеживаемые**, но при этом, при запросе 
статуса, `git` не напоминал нам о нем, можно добавить файл в список игнорируемых.

Для этого нужно создать еще один файл со специальным названием
```
.gitignorу
```
и в нем указать имя файла, который нужно игнорировать. После этого необходимо добавть 
файл `.gitignore` в отслеживаемые и заоммитить изменения.

## 6. Экранирование
Чтобы спецсимволы не исчезали и не влияли на оформление, нужн
```
\*\*жирный\*\* 
\*курсив\*

```
\*\*жирный\*\* -------- **жирный**

\*курсив\* -------- *курсив* 

# **Работа с удаленными репозиториями**
## 1. Регестрация на GITHub.
Первым делом необходимо зарегестрироваться на [GitHub](github.com)
## 2. Клонировапние репозитория
### a) Находим на просторах `github` репозиторий, изменения в который мы хотим внести.
### b) Используя кнопку `fork` создаем копию этого репозитория в свой аакаунт на `github` и переходим туда.
### c) С помощью кнопки `code` открываем ссылку на этот репозиторий и копируем ее.
### d) В `VSC` вводим комманду:
```
git clone *ссылка_на_репозиторий*
```
выбираем нужный нам файл и открываем его в терминале используя комманду 
```
cd
```
или нажав в проводнике на файл правой кнопкой мыши и выбрав `открыть во встроеном терминале`

## 3. Внесение изменений
### a) Файл, который мы хотим изменить, добвяем к отслежеванию, и создаем в нем новую ветку. 
### b) Переходим в ветку, вносим небходимы изменния, не забывая делать коммиты.
### c) По завершению работы пушим изменения с помощью комманда `git push` и следуем инструкциям git.
## 4. Создаем PULL REQUEST
Если все вышеперечисленные действия сделаны верно, то на `github` появится кнопка `compare & pull request`.

Нажимаем на нее, указываем какую ветку в форнутом репозитории мы предлагаем слить с исходным репозиторием.

Пишем заголовки и комментарии

И нажимаем `create pull request` 

после нажатия ждем появления зеленой галочки, означающей, что pull request отправлен.
