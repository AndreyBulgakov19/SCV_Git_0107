# Работа с GIT
## 1. Проверка наличия установленного Git
В терминале выполнить команду `Git version`
Если Git установлен, появится сообщение с версией программы. Иначе будет сообщение об ошибке

## 2. Установка Git
Загружаем последнюю версию Git с сайта:                                                             

 https://git-scm.com/downloads

Устанавливаем с настройками по умолчанию

## 3. Настройка Git
При первом использовании Git необходимо представиться. Для этого нужно ввести в терминале две команды 
```
 git config --global user.name "Ваше имя"
 git config --global user.email "Ваша почта@xample.com"
```

## 4. Создание репозитория
Получить репозиторий можно двумя способами.
1. В терминале переходим к папке, в которой хотим создать репозиторий. Выполняем команду:
```
Git init
```
Клонировать существующий репозиторий Git из любого места. Сделать это можно так:
```
git clon <адрес репозитория>
```
## 5. Запись изменений в репозиторий
Каждый файл в рабочей папке (репозитории) может находиться в одном из двух состояний: под версионным контролем (отслеживаемые - tracked) и нет (неотслеживаемые - untracked).

## 6. Просмотр истории коммитов
Для просмотра истории коммитов используется команда:
```
git log
```
Для выхода из истории режима просмотров истории используется клавиша **`"Q"`**

## 7. Проверка текущего состояния Git
Чтобы узнать текущее состояние Git и есть ли изменения, которые нужно сохранить существует команда:
```
git status
```
## 8. Добавление содержимого в индекс (staging area)
Добавитьсодержимое рабочего каталога виндекс (staging area) для последующего сохранения поможет комманда:
```
git add -m <Имя файла>
```

## 9. Сохранение
Сохранить все данные можно с помощью команды:
```
git commit -m "Сообщение"
```
Команда `git commit` берет все данные, добавленные в индекс с помощью команды `git add` и сохраняет их слепок во внутренней базе данных, а затем сдвигает указатель текущей ветки на этот слепок

## 10. Переключение между версиями
Перейти в любую из версий можно с помощью команды:
```
git checkout <Имя версии>
```
Для того чтобы вернуться в последнюю версию необхоимо ввести команду:
```
git checkout master
```

## 11. Просмотр разницы между файлами
Просмотреть разницу между текущим файлом и сохраненным поможет команда:
```
git diff
```

## 12. Создание новой ветки
Для создания новой ветки необходимо ввести команду:
```
git branch <имя новой ветки>
```

## 13. Слияние веток
Для слияния веток необходимо ввести команду:
```
git merge <имя ветки для слияния с текущей>
```

## 14. Удаление веток
Если новую ветку соединили с текущей, и ветка больше не нужна, то её можно удалить с помощью команды:
```
git branch -d <имя удаляемой ветки>
```
Если соединять ветку с текущей нет необходимости, то ее можно удалить с помощью команды:
```
git branch -D <имя удаляемой ветки>
```

## 15. Разрешение конфликтов
При работе в двух ветках одновременно может возникнуть ситуация, когда в одной и другой ветке мы по разномуизменили блок текста. Если затем мы попробуем слить эти ветки, Git сообщит о конфликте и предложит выбрать какие же изменения нужно записать

## 16. Копирование удаленного репозитория 
Чтобы скопировать удаленный репозиторий на свой ПК необходимо воспользоваться командой:
```
git clone <адрес репозитория>
```
Команда `git clone` составная: она не только загружает все изменения, но и пытается слиь все ветки на локальном компьютере и в удаленном репозитории.

## 17. Извлечение содержимого из удаленного репозитория
Для извлечения и загрузки содержимого из удаленного репозитория и немедленного обновления локального репозитория этим содержимым существует команда:
```
git pull
```

## 18. Выгрузка содержимого локального репозитория в удаленный репозиторий
Отправить свою версию репозитория во внешний репозиторий можно спомощью команды:
```
git push
```
Если выгрузка происходит впервые, то необходимо авторизоваться на удаленном репозитории.

## 19. Как сделать pull request

`pull request` - командадля предложения изменений в чужой удаленный репозиторий.
Для того чтобы сделать pull request необходимо:
1. делаем ответвление репозитория с помощью кнопки `fork`
2. делаем git clone **`своей версии`** репозитория
3. создаем новую ветку и в нее вносим изменения
4. фиксируем изменения (делаем коммиты)
5. отправляем свою версию в свой профиль GitHub
6. на сайте GitHub нажимаем кнопку pull request

## 20. Как настроить совместную работу

1. создать аккаунт на GitHub.com
2. создать локальный репозиторий в VSCode
3. "подружить" локальный и удаленный репозиторий. (GitHub при создании нового репозитория подскажет как это сделать)
4. создать новую ветку, произвести в ней изменения, и все закоммитить
5. отправить (push) локальный репозиторий в удаленый (на GitHub), при этом, возможно, нужно будет авторизоваться на удаленном репозитории
6. провести изменения с "другого компьютера"
7. выыкачать (pull) актуальное состояние из удаленного репозитория
